title: Jobs API
description: Create scrape jobs and retrieve their status and result.
---

## POST /api/jobs

Creates a new scrape job.

### Auth

Requires API key or authenticated admin session.

### Example Request

```bash
curl -X POST http://localhost:3001/api/jobs \
  -H "Content-Type: application/json" \
  -H "x-api-key: cx_your_api_key_here" \
  -d '{
    "url": "https://example.com",
    "strategy": "auto",
    "options": {
      "timeoutMs": 30000,
      "useCache": true,
      "cacheTtlSeconds": 900,
      "maxRetries": 2,
      "retryDelayMs": 1000,
      "waitUntil": "domcontentloaded"
    }
  }'
```

### Example Response

```json
{
  "jobId": "job_01hrx8qj2b1j6m3n9t1",
  "status": "queued",
  "queuedAt": "2026-04-11T07:10:25.000Z",
  "resultTtlSeconds": 3600
}
```

## GET /api/jobs/:jobId

Returns current job status.

### Example Request

```bash
curl http://localhost:3001/api/jobs/job_01hrx8qj2b1j6m3n9t1 \
  -H "x-api-key: cx_your_api_key_here"
```

### Example Response

```json
{
  "jobId": "job_01hrx8qj2b1j6m3n9t1",
  "status": "processing",
  "url": "https://example.com",
  "strategy": "auto",
  "requestedAt": "2026-04-11T07:10:25.000Z",
  "updatedAt": "2026-04-11T07:10:29.000Z",
  "fingerprint": "fp_2d5e2d...",
  "options": {
    "timeoutMs": 30000,
    "useCache": true
  }
}
```

## GET /api/jobs/:jobId/result

Returns the final job result when available.

### Example Request

```bash
curl http://localhost:3001/api/jobs/job_01hrx8qj2b1j6m3n9t1/result \
  -H "x-api-key: cx_your_api_key_here"
```

### Example Response

```json
{
  "jobId": "job_01hrx8qj2b1j6m3n9t1",
  "status": "completed",
  "url": "https://example.com",
  "strategy": "cloudscraper",
  "requestedAt": "2026-04-11T07:10:25.000Z",
  "completedAt": "2026-04-11T07:10:33.000Z",
  "preview": "<!doctype html><html><head><title>Example Domain</title>",
  "contentType": "text/html",
  "method": "GET",
  "responseTimeMs": 842,
  "retries": 0,
  "cached": false
}
```

## Request Body

```json
{
  "url": "https://example.com",
  "strategy": "auto",
  "options": {
    "timeoutMs": 30000,
    "method": "GET",
    "body": "optional raw body",
    "headers": {
      "User-Agent": "Crawlix Client"
    },
    "formData": {
      "key": "value"
    },
    "useCache": true,
    "cacheTtlSeconds": 900,
    "maxRetries": 2,
    "retryDelayMs": 1000,
    "waitUntil": "domcontentloaded",
    "waitForSelector": "#app",
    "waitForFunction": "window.__READY__ === true",
    "additionalDelayMs": 500,
    "useProxy": false
  }
}
```

## Notes

- `strategy` supports `auto`, `cloudscraper`, and `playwright`.
- `auto` is the recommended default.
- Status polling is the default client pattern in v1.
- Admin-only endpoints like retry and cancel are intentionally omitted from request examples in this docs section.
