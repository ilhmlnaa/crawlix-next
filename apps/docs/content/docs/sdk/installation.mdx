## Install

```bash
pnpm add @crawlixnext/sdk-node
```

## Requirements

- Node.js 18 or newer
- Crawlix Next API key
- Crawlix Next API base URL

## Minimal setup

```ts
import { CrawlixClient } from '@crawlixnext/sdk-node';

const client = new CrawlixClient({
  baseUrl: 'https://api.example.com',
  apiKey: 'cx_xxx',
});
```

## Client options

```ts
type CrawlixClientOptions = {
  baseUrl: string;
  apiKey: string;
  timeoutMs?: number;
  defaultHeaders?: Record<string, string>;
};
```

## Notes

- `baseUrl` should point to your Crawlix deployment origin, such as `https://crawlix-next-api.hamdiv.me`.
- The SDK automatically appends `/api` internally.
- Base URLs that already end with `/api` remain supported for backward compatibility.
- `apiKey` is sent as `x-api-key`.
- `defaultHeaders` is useful when you need to attach tenant or correlation metadata to every request.
