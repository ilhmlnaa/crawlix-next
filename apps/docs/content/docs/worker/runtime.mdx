## Worker Health Endpoints

### GET /worker/health/live

```bash
curl http://localhost:3002/worker/health/live
```

Example response:

```json
{
  "status": "ok"
}
```

### GET /worker/health/ready

```bash
curl http://localhost:3002/worker/health/ready
```

Example response:

```json
{
  "status": "ok",
  "service": "crawlix-worker",
  "redis": {
    "status": "ok"
  },
  "queue": {
    "status": "ok",
    "queueName": "crawlix.scrape.jobs",
```

## Runtime Notes

- The worker keeps browser runtime inside the worker process, not in the API.
- Playwright browser instances are reused through a lightweight pool.
- Worker heartbeat is stored in Redis so the dashboard can list active workers.
- `WORKER_ALLOWED_STRATEGIES` controls worker capability and can be set to `cloudscraper`, `playwright`, or both as a comma-separated list.
- Each worker also exposes a stable `workerId` for the current runtime and consumes a dedicated queue shaped like `crawlix.scrape.jobs.worker.<workerId>`.
- Jobs created with `targetWorkerId` are published only to that worker-specific queue. They do not fall back to the shared queue automatically.
- The admin dashboard can list active workers and use their `workerId` for targeted dispatch decisions.
- When a targeted job requests an incompatible strategy, the API rejects it before publish and the worker also enforces the same rule during execution.
- Terminal job events can be pushed to third-party systems through a dedicated webhook queue that is separate from the scrape execution queue.
- Webhook delivery uses its own retry and dead-letter queues, so webhook callback failures do not block scrape workers from finishing jobs.
- Workers sign outbound webhook payloads with `WEBHOOK_SIGNING_SECRET` unless the job explicitly carries its own `webhookSecret`.
- Proxy is resolved in worker runtime from each job options with this precedence:
  - if `useProxy` is `false`, no proxy is used.
  - if `useProxy` is `true` and `proxyUrl` is provided, worker uses that URL.
  - if `useProxy` is `true` and `proxyUrl` is empty, worker falls back to `SCRAPER_PROXY_URL`.
  - if both proxy values are empty, request runs direct.
